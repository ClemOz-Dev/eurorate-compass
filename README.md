# Barom√®tre BCE (personal rates forecaster)

This is a small personal web backend built with **Next.js**. It pulls macro/finance time series from public APIs, computes a **probabilistic forecast** for a few key rate metrics (currently: *French mortgage rates on new loans with initial rate fixation period > 10 years*), and exposes the result via JSON endpoints that a future UI can consume.

At this stage the project focuses on the **backend** (data fetching + forecasting). A UI (charts + ‚Äúnews feed‚Äù summaries) can be added later.

## What it does (high level)

1. Fetch a historical interest-rate series from **DBnomics** (which aggregates Banque de France data).
2. Fit a simple **AR(1)** model on the last N months of history.
3. Perform a **bootstrap simulation of residuals** to generate a distribution forecast.
4. Return forecast values for horizons **+12 / +24 / +36 months** as:
   - `median`
   - `low` quantile (~10%)
   - `high` quantile (~90%)

This is intentionally ‚Äúscenario probabilistic‚Äù: it provides a *range* of plausible outcomes, not a guaranteed future rate.

## Backend API (endpoints)

### Health check

`GET /api/health`

Returns `{ ok: true, time: ... }`.

File:
- `src/app/api/health/route.ts`

### Mortgage rates forecast (12/24/36 months)

`GET /api/forecast/mortgage-rates-fr`

Returns JSON including:

- The series metadata (provider/dataset/series code, last observation)
- The model fit (AR(1) parameters + residual sigma)
- Forecast horizons (`+12`, `+24`, `+36` months):
  - `period` (YYYY-MM)
  - `median`, `low`, `high`
- A disclaimer that this is a probabilistic statistical forecast, not a guarantee.

Optional query parameter:

- `fitWindow` (number of months used for fitting, default `60`, minimum enforced by model)

Implementation:
- `src/app/api/forecast/mortgage-rates-fr/route.ts`
- uses:
  - `src/lib/dbnomics.ts` (fetch historical series)
  - `src/lib/forecast/ar1-bootstrap.ts` (AR(1) bootstrap residuals)

### MVP combined dashboard (forecast + LLM synthesis)

`GET /api/dashboard`

Returns forecast quantiles plus an English ‚Äúactu‚Äù style synthesis generated by Mistral (server-side).

Implementation:
- `src/app/api/dashboard/route.ts`
- uses:
  - DBnomics fetch + AR(1) bootstrap (same as the forecast endpoint)
  - `src/lib/mistral.ts` (calls `POST https://api.mistral.ai/v1/chat/completions`)

Note:
- The endpoint now serves a persisted snapshot (`data/dashboard.snapshot.json`) by default, so UI refreshes do **not** trigger new LLM calls.
- If no snapshot exists yet, the endpoint builds one once, stores it, then serves it.
- Optional: `GET /api/dashboard?force=1` forces a rebuild (use sparingly).

### Monthly cron refresh (3rd day, 08:00)

`GET /api/cron/monthly-dashboard`

This route rebuilds the snapshot (DBnomics fetch + model + LLM synthesis) once and stores it.

Security:
- Requires header `Authorization: Bearer <CRON_SECRET>`.
- Set `CRON_SECRET` in environment variables.

Scheduling:
- `vercel.json` includes:
  - `0 8 3 * *` (UTC): every month, day 3, 08:00.
- On Vercel, this endpoint is called automatically by Vercel Cron.

## External APIs consumed

### DBnomics (aggregates Banque de France series)

The forecast endpoint uses the DBnomics API endpoint:

- `https://api.db.nomics.world/v22/series/{provider}/{dataset}/{series}?observations=true`

Series used (via DBnomics):

- Provider: `BDF`
- Dataset: `MIR1`
- Series code: `M.FR.B.A2C.P.R.A.2250U6.EUR.N`

Label (as returned by DBnomics):

> New loans for house purchase, households, over 10 years, annualised agreed rate

Note: the forecast is computed on the **annualised agreed interest rate** series (percent per year), not directly on the bank‚Äôs TAEG with insurance and fees.

## The forecasting method (AR(1) + bootstrap residuals)

In `src/lib/forecast/ar1-bootstrap.ts`:

- We estimate an AR(1) form on the last `fitWindow` months:
  - `y_t = c + phi*y_{t-1} + e_t`
- Residuals `e_t` are resampled with replacement (bootstrap).
- For each simulation:
  - we propagate the AR(1) forward month-by-month to produce future values
- Quantiles across simulations yield:
  - `median` (0.5)
  - `low` (0.1)
  - `high` (0.9)

## How to run locally

Prerequisites:

- Node.js + npm

Commands:

```bash
cd /path/to/barometre-bce
npm install
npm run dev -- --port 3100
```

Test the endpoints:

```bash
curl http://localhost:3100/api/health
curl http://localhost:3100/api/forecast/mortgage-rates-fr
```

## How to consume from a frontend

In your future UI, call the JSON endpoints:

- `GET /api/forecast/mortgage-rates-fr`

and draw charts from the `forecasts` object.

For ‚Äúactu‚Äù text synthesis + summaries, you would add a second step where the backend:

1. fetches news (RSS or another data source)
2. calls the LLM server-side (using `MISTRAL_API_KEY`)
3. stores the structured result

Then the frontend displays:

- charts from numeric series
- a news feed from the structured LLM summary

## Disclaimer

This project provides statistical estimates based on historical data. It is not financial advice. Future rates can deviate significantly due to regulatory changes, product design changes by banks, macro shocks, and changes in market expectations.
